# Design System Rules

Auto-UX integration rules for AI-assisted development. These rules ensure new and modified components stay aligned with the design system in `src/design-system/`.

---

## Component Selection Rules

- **Expandable/collapsible section** -> Use `ExpandableSection` primitive from `@/design-system`. Never inline `AnimatePresence` + `motion.div` + `height:'auto'`.
- **Copyable identifier** (order ID, tracking, FNSKU, serial, ticket) -> Use the semantically correct `CopyChip` variant. Never build inline clipboard buttons.
- **Tab navigation** -> Use `TabSwitch` component. Never build custom pill/button groups with manual active state.
- **Search input** -> Use `SearchBar` (visible), `CompactSearchInput` (inline), or `SearchField` (primitive). Never use raw `<input>` with custom search styling.
- **Search toggle** (icon that expands to input) -> Use `OverlaySearch` component. Never inline the AnimatePresence toggle pattern.
- **Data table date headers** -> Use `DateGroupHeader` component. Never inline sticky date group dividers.
- **Form field with label** -> Use `FormField` component. Never inline label + required indicator + hint.
- **Item condition display** -> Use `ConditionText` primitive or `getConditionColor`/`formatConditionLabel` helpers. Never hardcode condition-to-color mapping.
- **Status display** -> Use `StatusBadge` or `StatusMicroLabel`. Never inline status-to-color mapping objects.
- **Sticky positioning** -> Use `StickyHeader` primitive for consistent z-index and optional frosted glass.
- **Icon action rows** (copy, edit, external link) -> Use `ActionButtonGroup` primitive.

## Token Usage Rules

### Typography

Never hardcode these patterns -- use the preset from `@/design-system`:

| Pattern | Preset |
|---------|--------|
| `text-[10px] font-black uppercase tracking-[0.2em] text-gray-500` | `typographyPresets.sectionLabel` |
| `text-[10px] font-bold uppercase tracking-[0.16em] text-gray-700` | `typographyPresets.fieldLabel` |
| `text-[13px] font-bold text-gray-900` | `typographyPresets.dataValue` |
| `text-[13px] font-bold font-mono text-gray-900` | `typographyPresets.monoValue` |
| `text-[11px] font-extrabold font-mono` | `typographyPresets.chipText` |
| `text-base font-black text-gray-900 leading-tight` | `typographyPresets.cardTitle` |
| `text-[10px] font-black uppercase tracking-[0.16em] text-gray-500` | `typographyPresets.tableHeader` |
| `text-[13px] font-semibold text-gray-900` | `typographyPresets.tableCell` |

### Motion

Never hardcode transition duration/easing in `motion.div`. Use named presets:

- `framerTransition.stationCardMount` -- active order card
- `framerTransition.upNextRowMount` -- up-next list row
- `framerTransition.tableRowMount` -- table row enter/exit
- `framerTransition.sidebarExpand` -- sidebar section expand/collapse
- `framerTransition.dropdownOpen` -- dropdown menu
- `framerTransition.overlaySearchIn` -- overlay search toggle
- `framerTransition.chipCopyFeedback` -- copy feedback flash

Never hardcode `initial`/`animate`/`exit` in AnimatePresence children. Use named presets:

- `framerPresence.stationCard` / `framerPresence.upNextRow`
- `framerPresence.tableRow` -- simple opacity fade
- `framerPresence.dropdownPanel` -- fade + slide from top
- `framerPresence.sidebarSection` -- height expand/collapse
- `framerPresence.collapseHeight` -- generic height collapse

### Colors

Functional color mapping (never deviate):

| Domain | Color |
|--------|-------|
| Repair / Support | Orange |
| Inventory alert | Red |
| System identifiers | Gray |
| Logistics / Tracking | Blue |
| Success / Completion | Emerald / Green |
| Fulfillment channel | Purple |
| Queued / Pending | Yellow |

Item condition colors (use `getConditionColor` or `semanticColors.condition`):

| Condition | Color |
|-----------|-------|
| Used (default) | `text-black` |
| New | `text-yellow-500` |
| Parts | `text-amber-800` |
| Qty >= 2 | `text-yellow-500` |

### Spacing

Use density tiers for consistent spacing:

| Context | Density | Values |
|---------|---------|--------|
| Table rows, compact data | `density.compact` | px: 0.5rem, py: 0.375rem, gap: 0.375rem |
| Sidebar rows, cards | `density.standard` | px: 0.75rem, py: 0.5rem, gap: 0.5rem |
| Form fields, panels | `density.spacious` | px: 1rem, py: 0.75rem, gap: 0.75rem |

## Composition Rules

1. **Primitives** (`design-system/primitives/`) are atomic -- accept data and callbacks, never fetch data or manage global state.
2. **Components** (`design-system/components/`) compose primitives -- may manage local UI state (open/closed, hover, animation) but never fetch data.
3. **Feature components** (`src/components/`) compose design system components -- own data fetching, routing, and business logic.
4. When a pattern appears in 3+ feature components, extract it:
   - Purely visual -> primitive
   - Visual + local UI state -> component
   - Needs data/business logic -> keep in feature layer, extract visual part

## CopyChip Semantic Binding

**Hard rule: chip variants are bound to data types. Never interchange.**

| Chip | Color | Use For |
|------|-------|---------|
| `TrackingChip` | Blue | Carrier shipping tracking |
| `FnskuChip` | Purple | Amazon FNSKU |
| `SerialChip` | Emerald | Device serial numbers |
| `OrderIdChip` | Gray | Internal order IDs |
| `TicketChip` | Orange | Repair/support tickets |
| `SourceOrderChip` | Gray | External platform order numbers |

## Anti-Patterns

- Creating a new chip/badge variant without checking the CopyChip family
- Using `rounded-full` for non-circular elements (use `radii.full` only for pills and avatar circles)
- Inline color-to-className mapping objects (use `StatusBadge` or condition tokens)
- Duplicating AnimatePresence expand/collapse boilerplate (use `ExpandableSection`)
- Building custom dropdowns from scratch (use `ViewDropdown` or Headless UI)
- Using `setTimeout` for animation delays (use Framer Motion `transition.delay`)
- Hardcoding z-index values (use z-index ladder from tokens)

## File Organization

```
design-system/
  tokens/           -- raw values (colors, sizes, weights, spacings, motion)
  foundations/       -- computed from tokens (breakpoints, motion presets)
  themes/            -- token compositions for light/dark
  primitives/        -- atomic visual blocks (no state beyond hover/focus)
  components/        -- composed from primitives (local UI state ok)
  utils/             -- CSS variable helpers, theme factories
  index.ts           -- barrel export
```
